# ShoeTracker (iOS)

แอพ iOS สำหรับบันทึกระยะทางวิ่งแยกตามรองเท้าแต่ละคู่ — บันทึกวัน เวลา และระยะทางของแต่ละครั้งที่วิ่ง
พร้อมดึงข้อมูลอัตโนมัติจาก **Apple Health** และ **Strava** เพื่อรวมยอดระยะทางสะสมของรองเท้าแต่ละคู่

> โปรเจกต์นี้เป็น**ซอร์สโค้ด Xcode** ที่ต้อง build ด้วย Mac + Xcode เอง — เขียนและทดสอบไม่ได้ในสภาพแวดล้อมนี้เพราะเป็น Linux
> ไม่มี Xcode/Swift toolchain ให้ compile ได้ ผู้ใช้ต้องเปิดโปรเจกต์บน Mac เพื่อ build และติดตั้งลงเครื่องของตัวเอง

## ฟีเจอร์หลัก

- จัดการรายการรองเท้า (เพิ่ม/แก้ไข/ทำเครื่องหมายเลิกใช้/ลบ) พร้อมยอดระยะทางสะสมต่อคู่ (คล้ายหน้า "Your Gear" ของ Strava)
- บันทึกการวิ่งด้วยมือ: วันที่ เวลา ระยะทาง ระยะเวลา และหมายเหตุ ผูกกับรองเท้าที่เลือก
- เชื่อมต่อ **Apple Health**: ดึงข้อมูลการวิ่ง (workout ประเภท running) มาเป็นรายการที่ต้องเลือกรองเท้าเอง (เพราะ HealthKit ไม่มีข้อมูลเรื่องรองเท้า)
- เชื่อมต่อ **Strava**: ดึงรายชื่อ "Gear" (รองเท้า) มาสร้าง/จับคู่อัตโนมัติ และดึงกิจกรรมประเภทวิ่งพร้อม gear ที่ผูกไว้ใน Strava มาใส่ให้อัตโนมัติ
- กันข้อมูลซ้ำ: การวิ่งที่ซิงค์มาจาก Health/Strava จะถูกจดจำด้วย ID ภายใน ไม่ซิงค์ซ้ำแม้กดซิงค์หลายครั้ง

## โครงสร้างโปรเจกต์

```
ShoeTracker/
├── project.yml                 # นิยามโปรเจกต์ Xcode (ใช้กับ XcodeGen)
├── Config.xcconfig.example     # ตัวอย่างไฟล์ค่าคอนฟิก Strava (คัดลอกเป็น Config.xcconfig)
└── ShoeTracker/
    ├── App/                    # จุดเริ่มแอพและค่าคอนฟิก
    ├── Models/                 # โมเดลข้อมูล SwiftData: Shoe, RunEntry
    ├── Services/                # HealthKit, Strava OAuth/API, ตัวจัดการซิงค์, Keychain
    ├── Views/                   # หน้าจอ SwiftUI ทั้งหมด
    └── Resources/Assets.xcassets
```

ข้อมูลถูกเก็บในเครื่องด้วย **SwiftData** (iOS 17+) — ไม่ต้องมีเซิร์ฟเวอร์ของตัวเอง

## สิ่งที่ต้องมีก่อนเริ่ม

- Mac ที่ติดตั้ง **Xcode 15 ขึ้นไป**
- [XcodeGen](https://github.com/yonaskolb/XcodeGen) สำหรับสร้างไฟล์ `.xcodeproj` จาก `project.yml`
  ```bash
  brew install xcodegen
  ```
- Apple ID (ฟรีก็เพียงพอ) สำหรับติดตั้งแอพลงเครื่อง iPhone ของตัวเองผ่าน Xcode
- (ถ้าต้องการเชื่อม Strava) บัญชี Strava และแอพที่สมัครไว้ที่ https://www.strava.com/settings/api

## ขั้นตอนติดตั้งและรันครั้งแรก

1. ตั้งค่าไฟล์คอนฟิก Strava (ทำได้แม้ยังไม่อยากเชื่อม Strava ทันที — เว้นค่าว่างไว้ก่อนได้):
   ```bash
   cd ios-app/ShoeTracker
   cp Config.xcconfig.example Config.xcconfig
   ```
   แล้วเปิด `Config.xcconfig` ใส่ `STRAVA_CLIENT_ID` และ `STRAVA_CLIENT_SECRET` จากแอพที่สมัครไว้บน Strava (ดูหัวข้อด้านล่าง)

2. สร้างไฟล์โปรเจกต์ Xcode:
   ```bash
   xcodegen generate
   ```
   จะได้ไฟล์ `ShoeTracker.xcodeproj` ขึ้นมา (ไฟล์นี้ไม่ได้เก็บใน git — สร้างใหม่ได้ทุกครั้งด้วยคำสั่งนี้)

3. เปิด `ShoeTracker.xcodeproj` ด้วย Xcode
   - ไปที่แท็บ **Signing & Capabilities** ของ target `ShoeTracker` แล้วเลือก Team เป็น Apple ID ของคุณ
   - แก้ `PRODUCT_BUNDLE_IDENTIFIER` ใน `project.yml` ให้ไม่ซ้ำกับใคร (เช่น `com.<ชื่อคุณ>.shoetracker`) แล้วรัน `xcodegen generate` ใหม่ ถ้า Xcode แจ้งว่า bundle id ซ้ำ

4. เชื่อมต่อ iPhone กับ Mac แล้วกด Run (▶) เพื่อติดตั้งลงเครื่องจริง
   - แนะนำให้ทดสอบ Apple Health บนเครื่องจริง เพราะ Simulator ไม่มีข้อมูลการวิ่งจริงให้ดึง

## การเชื่อมต่อ Strava

1. ไปที่ https://www.strava.com/settings/api สร้างแอพใหม่ (ใส่ชื่อแอพอะไรก็ได้)
2. คัดลอก **Client ID** และ **Client Secret** มาใส่ใน `Config.xcconfig`
3. ตั้งค่า **Authorization Callback Domain** ตามที่ Strava กำหนด — แอพนี้ใช้ custom URL scheme `shoetracker://strava-auth` เป็น redirect URI ผ่าน endpoint สำหรับมือถือของ Strava (`/oauth/mobile/authorize`)
   ข้อกำหนดของ Strava เรื่อง callback domain อาจเปลี่ยนแปลงได้ตามช่วงเวลา — ให้ตรวจสอบขั้นตอนล่าสุดจาก [เอกสาร Strava API](https://developers.strava.com/docs/authentication/) หากเชื่อมต่อไม่สำเร็จในขั้นตอนนี้
4. ในแอพ ไปที่แท็บ **ตั้งค่า** แล้วกด "เชื่อมต่อ Strava" เพื่อล็อกอินและอนุญาตสิทธิ์

**ข้อควรระวัง:** Client Secret ของ Strava ถูกฝังไว้ในตัวแอพ (client-side) ซึ่งเหมาะกับการใช้งานส่วนตัว/ทดสอบเท่านั้น ไม่ควรแจกจ่ายแอพนี้ให้คนอื่นในวงกว้างโดยไม่ทำ token exchange ผ่านเซิร์ฟเวอร์กลางของตัวเอง

## การเชื่อมต่อ Apple Health

ไปที่แท็บ **ตั้งค่า** แล้วกด "เชื่อมต่อ Apple Health" ระบบจะขอสิทธิ์อ่านข้อมูล workout ประเภทวิ่งและระยะทาง

## การซิงค์ข้อมูลทำงานอย่างไร

กดปุ่มซิงค์ (มุมซ้ายบนของหน้ารองเท้า หรือในหน้าตั้งค่า) เพื่อ:

1. ดึง workout วิ่งใหม่จาก Apple Health ที่ยังไม่เคยดึงมาก่อน — จะไปอยู่ในหมวด **"การวิ่งที่ยังไม่ระบุรองเท้า"** ให้เลือกรองเท้าเอง (สูงสุด 200 รายการต่อครั้ง)
2. ดึงรายชื่อรองเท้าจาก Strava Gear มาสร้าง/จับคู่กับรองเท้าในแอพโดยอัตโนมัติ (จับคู่ตามชื่อ)
3. ดึงกิจกรรมวิ่งล่าสุดจาก Strava (สูงสุด 100 รายการต่อครั้ง) พร้อมผูกรองเท้าให้อัตโนมัติตาม gear ที่ตั้งไว้ใน Strava

## ข้อจำกัดที่ควรรู้

- ต้องมี Mac และ Xcode ถึงจะ build/ติดตั้งแอพได้ — โค้ดในรีโปนี้ build ในสภาพแวดล้อมนี้ (Linux) ไม่ได้
- การซิงค์ Strava ดึงกิจกรรมล่าสุดสูงสุด 100 รายการต่อการซิงค์หนึ่งครั้ง หากไม่ได้ซิงค์นานมากอาจต้องกดซิงค์ซ้ำหลายรอบ
- ยังไม่รองรับการส่งข้อมูลย้อนกลับไปยัง Apple Health หรือ Strava (ดึงข้อมูลเข้ามาทางเดียว)
- ไอคอนแอพ (AppIcon) ยังเป็นแค่โครงว่าง ต้องใส่รูปเองใน `Assets.xcassets`
